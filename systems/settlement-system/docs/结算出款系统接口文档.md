# 结算出款系统接口文档

## 1. 接口说明

### 1.1 批次管理

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/settlements/batches` | 创建结算批次 | 按天、按周期生成待出款结算任务 |
| `GET /api/settlements/batches` | 查询结算批次列表 | 查看批次状态、处理进度和异常情况 |
| `GET /api/settlements/batches/{batchNo}` | 查询结算批次详情 | 查看某个批次的结算范围、金额和结果 |

### 1.2 结算单管理

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `GET /api/settlements/orders` | 查询结算单列表 | 查看服务者/商家的应结、实结和审核状态 |
| `GET /api/settlements/orders/{settlementNo}` | 查询结算单详情 | 查看单笔结算的金额构成和扣减明细 |
| `POST /api/settlements/orders/{settlementNo}/audit` | 审核结算单 | 财务审核通过后进入出款阶段 |
| `POST /api/settlements/orders/{settlementNo}/reject` | 驳回结算单 | 结算异常、资料不全或金额有误时退回 |

### 1.3 出款管理

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/settlements/payouts` | 发起出款批次 | 将审核通过的结算单推送到银行或支付通道 |
| `GET /api/settlements/payouts` | 查询出款批次列表 | 查看出款成功、失败、处理中状态 |
| `POST /api/settlements/payouts/{payoutBatchNo}/retry` | 重试出款批次 | 针对失败批次进行人工重试或补发 |

### 1.4 事件消费

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/settlements/events/clearing/generated` | 消费清分生成事件 | 接收清分结果后自动生成结算单或待出款单 |

## 2. 统一返回

```json
{
  "code": "0",
  "message": "success",
  "data": {},
  "requestId": "REQ..."
}
```

## 3. 核心要求

1. 审核必须幂等，避免重复流转。
2. 出款必须幂等，避免重复打款。
3. 失败重试必须保留原记录和重试链路。

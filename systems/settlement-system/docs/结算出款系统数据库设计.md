# 结算出款系统数据库设计

## 1. 设计目标

`settlement-system` 承接清分结果，生成结算单、审核流、出款批次和出款跟踪数据。

## 2. 核心表

| 表名 | 用途 |
| --- | --- |
| `t_settlement_batch` | 结算批次主表 |
| `t_settlement_order` | 结算单主表 |
| `t_settlement_item` | 结算明细项 |
| `t_payout_batch` | 出款批次 |
| `t_payout_record` | 出款记录 |
| `t_settlement_audit_log` | 结算审核日志 |
| `t_settlement_event` | 结算事件 |

## 3. 表说明

### 3.1 `t_settlement_batch`

- `batch_no`
- `batch_date`
- `settlement_type`
- `batch_status`
- `total_count`
- `total_amount`

### 3.2 `t_settlement_order`

- `settlement_no`
- `batch_no`
- `target_type`
- `target_no`
- `amount_should_settle`
- `deduct_amount`
- `amount_net_settle`
- `settlement_status`

### 3.3 `t_settlement_item`

- `item_no`
- `settlement_no`
- `item_type`
- `item_amount`

### 3.4 `t_payout_batch`

- `payout_batch_no`
- `batch_no`
- `payout_channel`
- `payout_status`

### 3.5 `t_payout_record`

- `payout_no`
- `payout_batch_no`
- `settlement_no`
- `target_no`
- `payout_amount`
- `payout_status`

### 3.6 `t_settlement_audit_log`

- `audit_no`
- `settlement_no`
- `audit_action`
- `audit_result`
- `operator_name`

### 3.7 `t_settlement_event`

- `event_no`
- `event_type`
- `biz_no`
- `event_status`

## 4. 索引设计

1. `t_settlement_batch.batch_date`
2. `t_settlement_order.batch_no`
3. `t_settlement_order.target_no`
4. `t_payout_batch.batch_no`
5. `t_payout_record.settlement_no`
6. `t_settlement_audit_log.settlement_no`

## 5. 归档策略

1. 结算单永久保留
2. 出款记录永久保留
3. 审核日志永久保留
4. 事件表按月归档

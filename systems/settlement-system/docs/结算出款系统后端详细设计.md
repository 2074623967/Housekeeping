# 结算出款系统后端详细设计

## 1. 系统边界

`settlement-system` 负责结算单生成、审核、出款批次和出款跟踪，不负责清分计算。

## 2. 服务拆分

### 2.1 `SettlementBatchService`

- 批次生成
- 批次查询

### 2.2 `SettlementOrderService`

- 结算单生成
- 结算单查询
- 状态流转

### 2.3 `AuditService`

- 审核提交
- 审核通过
- 审核退回

### 2.4 `PayoutService`

- 出款批次创建
- 出款记录更新
- 出款重试

### 2.5 `SettlementEventService`

- 消费清分结果事件
- 生成待结算单

## 3. 技术原则

1. 结算单按批次生成
2. 审核通过后才能出款
3. 出款失败必须可重试
4. 所有结算动作留审计日志

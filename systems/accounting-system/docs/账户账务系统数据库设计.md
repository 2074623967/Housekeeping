# 账户账务系统数据库设计

## 1. 设计目标

`accounting-system` 承接支付成功后的账务入账、余额变化、冻结解冻、调账和流水查询，目标是支持可审计、可追溯、可扩展的账户体系。

## 2. 核心表

| 表名 | 用途 |
| --- | --- |
| `t_account_subject` | 账户主体，如用户、商家、服务者、平台 |
| `t_account` | 账户主表，记录账户状态和账户归属 |
| `t_account_balance` | 账户余额快照，记录可用、冻结、在途金额 |
| `t_account_ledger` | 账户流水，所有入账、出账、冻结、解冻、调账动作都落流水 |
| `t_account_freeze` | 冻结单，记录冻结/解冻原因与生命周期 |
| `t_account_adjustment` | 调账单，记录人工调账和系统补偿 |
| `t_account_event` | 账户事件，承接下游支付、清分、结算事件 |

## 4. 索引设计

1. `t_account_subject.subject_type`
2. `t_account.subject_id`
3. `t_account_balance.account_no`
4. `t_account_ledger.account_no + created_at`
5. `t_account_ledger.biz_no`
6. `t_account_freeze.account_no + freeze_status`
7. `t_account_adjustment.account_no + adjust_status`
8. `t_account_event.biz_no + event_type`

## 5. 归档策略

1. 账户主表长期保留
2. 流水表按月分区或按月归档
3. 事件表保留最近 1 年热数据
4. 冻结与调账单据保留全量审计数据

## 6. 说明

后续第二阶段实现时，所有账户余额变化必须先落流水，再更新余额快照，保证账实一致和可审计。

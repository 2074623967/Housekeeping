# 账户账务系统接口文档

## 1. 接口说明

### 1.1 主体管理

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/accounting/subjects` | 新增账户主体 | 新用户、商家、服务者、平台主体入账前建档 |
| `GET /api/accounting/subjects` | 查询主体列表 | 主体检索、运营维护、财务核对 |
| `GET /api/accounting/subjects/{subjectId}` | 查询主体详情 | 查看某主体的状态、类型、归属信息 |

### 1.2 账户管理

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/accounting/accounts/open` | 开立账户 | 为主体创建可用、冻结、在途余额账户 |
| `GET /api/accounting/accounts` | 查询账户列表 | 按主体、类型、状态检索账户 |
| `GET /api/accounting/accounts/{accountNo}` | 查询账户详情 | 查看账户归属、状态、币种、余额信息 |
| `POST /api/accounting/accounts/{accountNo}/close` | 关闭账户 | 账户注销、异常处理、业务终止 |

### 1.3 余额操作

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/accounting/balances/credit` | 余额入账 | 支付成功、退款到账、补贴发放 |
| `POST /api/accounting/balances/debit` | 余额出账 | 提现、扣款、服务费划转 |
| `POST /api/accounting/balances/freeze` | 余额冻结 | 预占金额、争议款、待确认资金 |
| `POST /api/accounting/balances/unfreeze` | 余额解冻 | 冻结解除、支付撤销、审核通过 |

### 1.4 流水查询

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `GET /api/accounting/ledgers` | 查询流水列表 | 对账、审计、资金追踪 |
| `GET /api/accounting/ledgers/{ledgerNo}` | 查询流水详情 | 查看某一笔资金变更的完整轨迹 |

### 1.5 冻结调账

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/accounting/freezes` | 创建冻结单 | 创建冻结原因、金额、业务单关联 |
| `POST /api/accounting/freezes/{freezeNo}/unfreeze` | 冻结单解冻 | 释放冻结金额并恢复可用余额 |
| `POST /api/accounting/adjustments` | 创建调账单 | 人工修正、系统补偿、差错处理 |
| `POST /api/accounting/adjustments/{adjustNo}/approve` | 审批调账单 | 财务审核后让调账生效 |

### 1.6 事件消费

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/accounting/events/payments/success` | 消费支付成功事件 | 支付核心推送成功支付结果后自动入账 |
| `POST /api/accounting/events/clearing/generated` | 消费清分结果事件 | 清分系统推送分账/应收结果后生成账务动作 |

## 2. 统一返回

建议统一为：

```json
{
  "code": "0",
  "message": "success",
  "data": {},
  "requestId": "REQ..."
}
```

## 3. 关键约束

1. 所有入账接口必须带幂等键，避免重复记账。
2. 所有流水查询必须支持分页和时间范围过滤。
3. 所有冻结/解冻必须记录操作人、操作来源和原因。
4. 所有事件消费必须支持重复投递和乱序到达。

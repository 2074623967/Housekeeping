# 账户账务系统前端页面与交互说明

## 1. 页面范围

### 1.1 admin-web

1. 账户主体列表
2. 账户列表
3. 余额详情页
4. 账户流水页
5. 冻结单列表页
6. 调账单列表页
7. 账户事件页

## 2. 页面职责

### 2.1 账户主体列表

- 查看主体类型、主体名称、状态
- 支持开立账户入口

### 2.2 账户列表

- 查看账户号、主体、账户类型、余额状态
- 支持冻结、解冻、调账、详情查看

### 2.3 余额详情页

- 展示可用余额、冻结余额、在途余额
- 展示最近变动流水

### 2.4 账户流水页

- 按账户号、业务单号、日期筛选
- 展示借贷方向、前后余额、业务类型

### 2.5 冻结单列表页

- 查看冻结状态、冻结原因、解冻状态

### 2.6 调账单列表页

- 查看调账原因、审批状态和生效状态

## 3. 交互要求

1. 列表页必须支持搜索、重置、分页
2. 详情页必须支持抽屉或侧边栏展示
3. 冻结、解冻、调账必须二次确认
4. 所有操作必须带操作日志留痕

## 4. 设计原则

1. 页面风格与 `payment-core` 保持一致
2. 财务数据必须前置关键数字和状态标签
3. 表格必须支持高密度信息展示
4. 风险操作必须明显标识

# 账户账务系统后端详细设计

## 1. 系统边界

`accounting-system` 负责账户、余额、流水、冻结、解冻、调账，不直接负责支付通道，不直接负责清分结算结果生成。

## 2. 服务拆分

### 2.1 `AccountSubjectService`

- 主体创建
- 主体查询
- 主体状态变更

### 2.2 `AccountService`

- 账户开立
- 销户
- 账户状态变更

### 2.3 `BalanceService`

- 余额入账
- 余额出账
- 余额冻结
- 余额解冻

### 2.4 `LedgerService`

- 流水写入
- 流水查询
- 流水归档

### 2.5 `FreezeService`

- 冻结单生成
- 解冻单生成
- 冻结状态流转

### 2.6 `AdjustmentService`

- 调账单生成
- 调账审批
- 调账生效

### 2.7 `EventService`

- 消费支付成功事件
- 消费清分结果事件
- 消费结算补偿事件

## 3. 技术原则

1. 所有余额变更必须幂等
2. 所有流水写入必须先写后更
3. 所有资金操作必须保留原始业务单号
4. 所有补偿操作必须可追溯

## 4. 推荐接口前缀

- `/api/accounting/subjects`
- `/api/accounting/accounts`
- `/api/accounting/balances`
- `/api/accounting/ledgers`
- `/api/accounting/freezes`
- `/api/accounting/adjustments`
- `/api/accounting/events`

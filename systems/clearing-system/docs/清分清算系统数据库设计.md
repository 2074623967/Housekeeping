# 清分清算系统数据库设计

## 1. 设计目标

`clearing-system` 承接支付成功后的交易拆分、平台抽佣、渠道费用、服务者应收和商家应收，输出可结算的清分结果。

## 2. 核心表

| 表名 | 用途 |
| --- | --- |
| `t_clearing_batch` | 清分批次主表 |
| `t_clearing_order` | 单笔交易清分结果 |
| `t_clearing_rule` | 清分规则配置 |
| `t_clearing_fee_item` | 费用项明细 |
| `t_clearing_share_item` | 分账明细 |
| `t_clearing_event` | 清分事件表 |
| `t_clearing_snapshot` | 清分结果快照 |

## 3. 表说明

### 3.1 `t_clearing_batch`

- `batch_no`
- `batch_date`
- `batch_status`
- `source_type`
- `total_order_count`
- `total_amount`

### 3.2 `t_clearing_order`

- `clearing_no`
- `batch_no`
- `payment_order_id`
- `order_no`
- `merchant_amount`
- `worker_amount`
- `platform_amount`
- `channel_fee_amount`
- `clearing_status`

### 3.3 `t_clearing_rule`

- `rule_no`
- `rule_name`
- `rule_type`
- `rule_expression`
- `rule_status`

### 3.4 `t_clearing_fee_item`

- `fee_item_no`
- `clearing_no`
- `fee_type`
- `fee_amount`
- `fee_bearer`

### 3.5 `t_clearing_share_item`

- `share_item_no`
- `clearing_no`
- `share_type`
- `share_amount`
- `share_target_no`

### 3.6 `t_clearing_event`

- `event_no`
- `event_type`
- `biz_no`
- `event_status`

### 3.7 `t_clearing_snapshot`

- `snapshot_no`
- `clearing_no`
- `snapshot_payload`
- `created_at`

## 4. 索引设计

1. `t_clearing_batch.batch_date`
2. `t_clearing_order.batch_no`
3. `t_clearing_order.payment_order_id`
4. `t_clearing_rule.rule_type`
5. `t_clearing_fee_item.clearing_no`
6. `t_clearing_share_item.clearing_no`
7. `t_clearing_event.biz_no`

## 5. 归档策略

1. 清分批次与结果永久保留
2. 事件表按月归档
3. 快照表支持审计回放

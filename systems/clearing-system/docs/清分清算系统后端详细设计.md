# 清分清算系统后端详细设计

## 1. 系统边界

`clearing-system` 只负责清分规则执行、费用计算、分账结果生成，不负责支付收单和出款执行。

## 2. 服务拆分

### 2.1 `ClearingRuleService`

- 清分规则加载
- 规则版本管理

### 2.2 `ClearingBatchService`

- 批次生成
- 批次重跑
- 批次查询

### 2.3 `ClearingOrderService`

- 单笔清分计算
- 分账结果落库

### 2.4 `FeeService`

- 平台抽佣
- 渠道手续费
- 活动补贴

### 2.5 `ShareService`

- 服务者分账
- 商家分账
- 平台收益汇总

### 2.6 `ClearingEventService`

- 消费支付成功事件
- 生成待结算清分结果

## 3. 技术原则

1. 支付单清分只能执行一次
2. 重跑必须生成新批次或新版本
3. 每笔金额必须可拆解到费用项
4. 所有结果支持审计追溯

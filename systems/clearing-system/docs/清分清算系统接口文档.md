# 清分清算系统接口文档

## 1. 接口说明

### 1.1 批次管理

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/clearing/batches` | 创建清分批次 | 按天、按小时、按业务线发起清分任务 |
| `GET /api/clearing/batches` | 查询清分批次列表 | 查看批次进度、状态、处理结果 |
| `GET /api/clearing/batches/{batchNo}` | 查询批次详情 | 查看单个批次的订单范围和结果统计 |
| `POST /api/clearing/batches/{batchNo}/rerun` | 重新运行批次 | 批次失败后人工重跑或补偿重算 |

### 1.2 清分结果

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `GET /api/clearing/orders` | 查询清分明细列表 | 查看每笔订单的应分、实分、差额 |
| `GET /api/clearing/orders/{clearingNo}` | 查询清分明细详情 | 排查某笔清分结果的计算过程 |

### 1.3 规则管理

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `GET /api/clearing/rules` | 查询清分规则列表 | 查看当前生效的清分、分账、扣费规则 |
| `POST /api/clearing/rules` | 新增清分规则 | 配置新业务线或新合作方规则 |
| `POST /api/clearing/rules/{ruleNo}/enable` | 启用规则 | 发布前的规则上线 |
| `POST /api/clearing/rules/{ruleNo}/disable` | 停用规则 | 规则下线、异常止损、业务切换 |

### 1.4 费用与分账

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `GET /api/clearing/fees` | 查询费用规则 | 查看渠道费、平台服务费、补贴费 |
| `POST /api/clearing/fees` | 新增费用规则 | 配置渠道费率、固定费用、封顶规则 |
| `GET /api/clearing/shares` | 查询分账明细 | 查看平台、商家、服务者的分账结果 |

### 1.5 事件消费

| 接口 | 说明 | 典型用途 |
| --- | --- | --- |
| `POST /api/clearing/events/payments/success` | 消费支付成功事件 | 收到支付成功后触发清分入队或直接清分 |

## 2. 返回规范

统一返回：

```json
{
  "code": "0",
  "message": "success",
  "data": {},
  "requestId": "REQ..."
}
```

## 3. 核心要求

1. 清分批次必须带幂等键，避免重复创建批次。
2. 单笔支付清分必须支持重复消费，避免重复分账。
3. 重跑清分必须保留历史版本，便于审计与追溯。

# gateway-access 后端详细设计

## 1. 目标

提供支付网关接入治理能力，包括应用、渠道、证书和权限。

## 2. 设计原则

1. 接入治理与支付交易解耦
2. 配置优先，当前已使用 MyBatis + H2 持久化实现，后续可平滑切换 MySQL
3. 输出统一接口返回结构
4. 为 `payment-core` 提供真实接入边界

## 3. 第一版实现

1. MyBatis 持久化仓储
2. 统一总览接口
3. 列表查询接口
4. 启停接口

## 4. 后续扩展

1. MyBatis + MySQL 生产化切换
2. 证书版本治理
3. 灰度发布
4. 调用方审计
5. 限流与签名策略

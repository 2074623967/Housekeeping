本文档是 `payment-core` 在 `Sunday, July 19, 2026` 的一期接口与数据设计文档。

适用对象：

- 前端开发

### 2.1 第一阶段A：支付后台运营骨架

1. 支付运营工作台

3. 支付单管理查询
4. 退款中心入口查询
5. 服务者结算入口查询

## 3. 页面与接口映射

### 3.1 第一阶段A 页面与接口

| 页面 | 路由 | 接口 | 说明 |
| --- | --- | --- | --- |
| 工作台 | `/dashboard` | `GET /api/dashboard/summary` | 查询今日核心经营与风险指标 |
| 订单中心 | `/orders` | `GET /api/orders` | 查询订单列表 |
| 支付单管理 | `/payments` | `GET /api/payments` | 查询支付单列表 |
| 退款单管理 | `/refunds` | `GET /api/refunds` | 查询退款单列表，当前为退款中心预留入口 |
| 服务者结算单 | `/worker-settlements` | `GET /api/settlements/workers` | 查询服务者结算列表，当前为清结算系统预留入口 |

### 3.2 第一阶段B 页面与接口

| 页面/能力 | 路由或接口 | 说明 |
| --- | --- | --- |
| 收银台 | `/cashier/:prepayOrderNo` | 组织支付参数、展示支付方式 |
| 支付结果页 | `/payment-result/:paymentOrderId` | 展示支付成功/失败/处理中 |
| 支付单详情页 | `/payments/:paymentOrderId` | 后台查看支付详情 |
| 回调日志页 | `/payments/:paymentOrderId/notifies` | 后台查看渠道回调收口 |

## 4. 接口统一约定

当前接口统一返回：

```json
{
  "code": "0",
  "message": "success",
  "data": {},
  "requestId": "REQ202607190001"
}
```

说明：

- `code`：业务响应码，`0` 表示成功
- `message`：接口说明
- `data`：真实业务数据
- `requestId`：链路追踪号

## 2. 一期开发目标

一期目标不是只做演示骨架，而是先完成 `支付核心系统 MVP` 的正式底座，做出一个可运行、可联调、可测试、可继续扩展的支付核心域。

一期交付范围：

- 后端 Spring Boot 工程初始化
- 前端 Vue3 工程初始化
- 基础导航和后台布局
- 工作台页面
- 订单中心页面
- 支付单管理页面
- 退款入口页面
- 服务者结算入口页面
- 对应后端查询接口
- MySQL + MyBatis 持久化能力
- 支付主链路第一阶段B 的文档、接口、测试基线

## 3. 一期模块拆分

### 3.1 前端

- `DashboardView`：工作台
- `OrdersView`：订单中心
- `PaymentsView`：支付单管理
- `RefundsView`：退款中心预留入口
- `WorkerSettlementsView`：服务者结算预留入口

### 3.2 后端

- `DashboardController`
- `OrderController`
- `PaymentController`
- `RefundController`
- `SettlementController`

## 4. 一期数据策略

- 当前以 MySQL 为主数据源，所有核心查询接口与支付主链路接口都围绕数据库对象落地。
- 后端统一采用 `controller -> service -> serviceimpl -> mapper -> mapper.xml -> entity` 正式结构。
- 业务逻辑集中在 `serviceimpl`，数据库表实体统一沉淀在 `entity`。

## 5. 产品对标要求

一期实现过程中，需持续参考市面成熟支付产品的共性做法，至少覆盖：

1. 收银台参数组织
2. 支付单状态机
3. 回调收口与幂等
4. 后台详情追踪能力
5. 主动查单与超时关闭

## 6. 测试交付要求

一期代码提交前必须同步具备以下测试交付物：

1. 接口测试清单
2. 主流程测试用例
3. 异常流测试用例
4. 回归测试清单
5. 联调验收说明

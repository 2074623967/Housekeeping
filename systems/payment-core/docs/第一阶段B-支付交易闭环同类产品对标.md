# 第一阶段B-支付交易闭环同类产品对标

## 1. 文档目标

本文档用于从资深产品经理和资深财务架构师视角，对 `payment-core` 第一阶段B 的设计基线进行外部同类产品能力对标。

## 2. 对标关注点

支付核心系统在行业成熟方案里，通常都会重点关注以下能力：

1. 订单、账单、预付单、支付单对象分层
2. 收银台参数标准化组织
3. 多渠道支付方式展示与路由扩展
4. 异步回调幂等收口
5. 主动查单和超时关闭补偿
6. 支付轨迹、回调轨迹、事件轨迹可审计
7. 后台可运营、可排查、可追踪

## 3. 当前系统应补齐的成熟能力

### 3.1 产品侧

1. 支付详情页要同时展示基础信息、支付轨迹、回调轨迹、事件轨迹。
2. 收银台页面要有支付中、支付成功、支付失败三类结果页。
3. 后台操作必须有明确反馈，不能只做按钮不做结果说明。

### 3.2 财务侧

1. 账单状态与支付状态分离。
2. 回调日志与业务状态分离。
3. 所有关键资金动作都要可追溯。

### 3.3 技术侧

1. 支付提交、回调、查单、关闭都应具备幂等设计。
2. ServiceImpl 负责业务编排，Mapper 只负责数据访问。
3. 核心链路要支持后续扩展为任务化、告警化、审计化。

## 4. 本阶段落地要求

第一阶段B 至少达到以下标准：

1. 可演示完整支付闭环。
2. 可查看支付详情和关键轨迹。
3. 可主动查单和关闭支付单。
4. 可为后续账务、清分、结算系统提供事件基础。

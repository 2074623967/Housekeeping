# 第一阶段B-支付交易闭环需求清单

## 1. 文档定位

本文档当前作为 `payment-core` 一期支付交易闭环 V1 的需求冻结清单，区分：

1. 已落地需求
2. 仍需增强需求
3. 后续系统承接需求

## 2. 已落地需求

### 2.1 业务需求

1. 订单可发起支付
2. 支付过程可创建账单
3. 账单可生成预付单
4. 用户可在收银台完成支付提交
5. 回调可更新支付状态
6. 后台可查看支付轨迹

### 2.2 产品需求

1. 收银台展示订单摘要和应付金额
2. 收银台展示支付方式
3. 支付结果页可展示成功、处理中、关闭态
4. 后台可查看支付单与支付详情
5. 后台可执行查单、回调、关闭操作

### 2.3 技术需求

1. 独立数据库已拆出
2. MyBatis + mapper.xml 已落地
3. `serviceimpl` 已承担业务编排
4. 支付事件、回调日志、路由日志已落库

## 3. 仍需增强需求

### P0

1. `PC / H5 / APP` 收银台页面矩阵补齐到参考稿口径
2. 支付请求管理、支付处理日志、支付记录详情的前后端联动增强
3. 提交支付、回调、查单、关闭支付的异常码与失败态展示增强

本批已完成后端启动验证、收款记录接口验证、超时关单验证、回调幂等验证、查询条件下推与分页改造，以及配置化路由接入主链路。

### P1

1. 真实渠道查单
2. 渠道返回码映射与渠道错误码标准化
3. 路由健康度熔断、权重和更复杂规则表达式

本批已完成回调验签、成功/关闭状态迟到回调拦截、支付超时巡检任务和路由失败兜底。

### P2

1. 支付事件统一发布协议
2. 支付监控与告警能力细化
3. 集成测试、契约测试和端到端自动化测试补齐

## 4. 后续系统承接需求

以下需求不再由 `payment-core` 本期承接，而由后续系统继续展开：

1. 账务分录与总分账
2. 清分清算
3. 商家结算与服务者结算
4. 退款闭环
5. 对账差异闭环
6. 保证金扣罚闭环

## 5. 结论

这份需求清单当前已经从“待开发列表”转成“冻结版能力清单 + 后续增强清单”，可以直接作为下一步开发排期和测试排期输入。

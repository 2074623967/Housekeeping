# 第一阶段B-支付交易闭环需求清单

## 1. 文档定位

本文档用于承接 `第一阶段B-支付交易闭环设计` 中识别出的具体需求清单。

从 `Sunday, July 19, 2026` 开始，`payment-core` 的第一阶段B 开发必须严格按本需求清单逐项推进。

## 2. 需求总目标

目标：

把家政订单支付从 `订单创建` 到 `支付回调收口` 的主链路补齐，并具备支付后台可观测、可查单、可补偿、可关闭能力。

## 3. 需求分类

### 3.1 业务需求

1. 订单创建后生成账单
2. 账单可发起预付单
3. 用户可在收银台完成支付
4. 支付成功后回写订单状态
5. 支付失败后可关闭或重试
6. 回调缺失时可主动查单

### 3.2 产品需求

1. 收银台展示订单摘要和应付金额
2. 收银台展示可用支付方式
3. 支付中状态页可引导用户等待结果
4. 支付成功页可展示支付结果和后续动作
5. 支付失败页可展示失败原因和重试入口
6. 后台可查看支付单、尝试单、回调日志

### 3.3 技术需求

1. 支付单幂等
2. 回调幂等
3. 查单补偿
4. 渠道路由
5. 状态机收敛
6. 事件驱动扩展点

### 3.4 测试需求

1. 正常支付路径测试
2. 失败重试测试
3. 回调重复测试
4. 回调丢失测试
5. 超时关闭测试
6. 查单与回调冲突测试

## 4. 优先级

### P0

1. 订单发起支付
2. 预付单生成
3. 收银台参数组装
4. 支付提交
5. 渠道回调收口

### P1

1. 主动查单
2. 支付关闭
3. 支付单详情
4. 回调日志

### P2

1. 支付事件发布
2. 支付单巡检任务
3. 统一告警

## 5. 需求进入开发的顺序

1. 先完成接口与数据设计
2. 再完成前端页面设计
3. 再完成后端详细设计
4. 再写代码
5. 再测试
6. 再提交 Git

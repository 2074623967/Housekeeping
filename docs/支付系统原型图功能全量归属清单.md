# 支付系统原型图功能全量归属清单

## 1. 文档定位

本文档根据用户提供的两张支付系统原型截图整理，用于确认截图中的菜单和功能是否需要建设，以及明确每项功能的目标系统归属。

结论：

1. 截图中的功能都属于支付资金平台建设范围，不能因为当前 `payment-core` 尚未实现就删除。
2. 截图中的功能不应全部放在 `payment-core`。
3. 后续按 `payment-core`、`accounting-system`、`settlement-system`、`refund-center`、`reconciliation-system`、`wallet-system`、`gateway-access`、`ops-config-system`、`risk-control-system` 分系统建设。

## 2. 截图功能总览

### 2.1 收款、付款与退款

| 原型功能 | 目标系统 | 当前状态 | 建设说明 |
| --- | --- | --- | --- |

| 微信支付记录 | `payment-core` | 未独立实现 | 作为支付渠道维度查询和渠道结果管理 |
| 余额支付记录 | `wallet-system` | 未实现 | 余额扣款和钱包流水不能由 payment-core 直接承载 |
| 银行卡支付记录 | `payment-core` | 未独立实现 | 作为银行卡渠道支付记录和渠道流水查询 |
| 付款管理 | `settlement-system` | 未实现 | 对外付款、代付和出款批次管理 |

| 统一退款记录 | `refund-center` | 未实现 | 统一管理原路退款、退转付和退款结果 |
| 财务退款记录 | `refund-center` + `accounting-system` | 未实现 | 退款业务记录与财务账务结果分开管理 |

## 3. 与当前 payment-core 的关系

### 3.1 应继续保留在 payment-core

1. 收款管理
2. 统一支付记录
3. 微信支付记录
4. 银行卡支付记录
5. 收银台会话
6. 支付请求管理
7. 支付处理日志
8. 正向消费支付主链路

### 3.2 只保留入口，不承载完整作业

以下内容可以在 `payment-core` 保留只读入口或业务关联入口，但完整作业必须由其他系统承载：

1. 退款管理 -> `refund-center`
2. 服务者结算查询 -> `settlement-system`
3. 商家余额查询 -> `wallet-system`
4. 分账记录 -> `clearing-system`
5. 收款认领 -> `reconciliation-system`

### 3.3 不应继续放入 payment-core

1. 付款管理、三方代付、监管付款、采购付款、批量付款
2. 账户管理、开销户管理、二清合规
3. 充值、提现、钱包业务、钱包账户管理、钱包交易记录
4. 支付信息管理、业务线管理、应用来源管理、接入权限
5. 支付渠道管理、证书管理、银行卡限额管理

## 4. 真实建设顺序

截图功能纳入总体建设范围后，后续顺序调整为：

1. 完成 `payment-core` 收款类能力增强
2. 建设 `refund-center`
3. 建设 `accounting-system`
4. 建设 `clearing-system`
5. 建设 `settlement-system`
6. 建设 `reconciliation-system`
7. 建设 `wallet-system`
8. 建设 `gateway-access`
9. 建设 `ops-config-system`
10. 建设 `risk-control-system`

## 5. 最终判断

这两张图片里的功能需要做，而且应作为支付平台的正式需求基线。

但“需要做”不等于“全部在 payment-core 里做”。专业拆分方式是：

- `payment-core` 负责收款和正向支付交易
- `refund-center` 负责退款
- `accounting-system` 负责账户账务
- `clearing-system` 负责分账清分
- `settlement-system` 负责付款出款
- `reconciliation-system` 负责收款认领和对账
- `wallet-system` 负责钱包、充值、提现和余额
- `gateway-access` 负责接入、权限、证书和渠道技术接入
- `ops-config-system` 负责支付配置和运营管理
- `risk-control-system` 负责限额、二清合规和风险控制

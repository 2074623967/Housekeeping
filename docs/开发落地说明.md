## 2. 一期开发目标

一期目标不是一次性做完整支付平台，而是先完成 `支付核心系统 MVP`，做出一个可运行、可演示、可继续扩展的骨架工程。

一期交付范围：

- 后端 Spring Boot 工程初始化
- 前端 Vue3 工程初始化
- 基础导航和后台布局
- 工作台页面
- 订单中心页面
- 支付单管理页面
- 退款入口页面
- 服务者结算入口页面
- 对应后端查询接口
- 本地 mock 数据服务

## 3. 一期模块拆分

### 3.1 前端

- `DashboardView`：工作台
- `OrdersView`：订单中心
- `PaymentsView`：支付单管理
- `RefundsView`：退款中心预留入口
- `WorkerSettlementsView`：服务者结算预留入口

### 3.2 后端

- `DashboardController`
- `OrderController`
- `PaymentController`
- `RefundController`
- `SettlementController`

## 4. 一期数据策略

- 当前先采用内存假数据，确保前后端联调链路跑通。
- 第二阶段再逐步引入数据库、状态流转、审批、账务和对账能力。

## 5. 下一阶段方向

二期开始再进入：

- 账户体系
- 账务流水
- 清分清算
- 服务者结算
- 商家结算

三期再进入：

- 退款闭环
- 对账工作台
- 保证金
- 配置中心

四期再进入：

- 钱包与红包
- 线下汇入
- 银行流水
- 二清监管
- 业财税票协同

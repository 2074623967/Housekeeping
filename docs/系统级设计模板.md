# 系统级设计模板

## 1. 文档定位

本文档是 `home-service-payment-system` 的系统级设计模板。

从 `Sunday, July 19, 2026` 开始，后续每个系统都必须按本模板输出完整设计，不再接受只写零散方案或只写部分接口的做法。

适用系统包括但不限于：

- `payment-core`
- `accounting-system`
- `clearing-system`
- `settlement-system`
- `refund-center`
- `reconciliation-system`
- `deposit-system`
- `wallet-system`

## 2. 必填章节

每个系统至少要包含以下章节：

1. 业务背景
2. 建设目标
3. 系统边界
4. 上下游关系
5. 核心业务流程
6. 核心对象模型
7. 状态机与状态流转
8. 数据库设计
9. 前端页面设计
10. 后端服务设计
11. 接口文档
12. 异常处理与补偿机制
13. 风险控制与审计要求
14. 排期与 Git 提交节点

## 3. 模板正文

### 3.1 业务背景

- 当前业务场景是什么
- 为什么要建设这个系统
- 当前痛点是什么

### 3.2 建设目标

- 一句话目标
- 阶段目标
- 最终目标

### 3.3 系统边界

- 系统负责什么
- 系统不负责什么
- 与其他系统的边界在哪里

### 3.4 上下游关系

- 上游系统
- 下游系统
- 同级协同系统

### 3.5 核心业务流程

至少输出：

- 正向流程
- 逆向流程
- 异常流程

### 3.6 核心对象模型

至少说明：

- 主单据
- 子单据
- 流水对象
- 事件对象

### 3.7 状态机与状态流转

至少说明：

- 初始状态
- 中间状态
- 终态
- 可逆状态
- 不可逆状态

### 3.8 数据库设计

至少输出：

- 表清单
- 表用途
- 字段字典
- 索引设计
- 归档策略

### 3.9 前端页面设计

至少输出：

- 端划分
- 页面清单
- 页面职责
- 查询区/列表区/详情区
- 页面交互

### 3.10 后端服务设计

至少输出：

- controller
- service
- repository
- dto
- enums
- job/task

### 3.11 接口文档

至少输出：

- 接口清单
- 请求参数
- 返回参数
- 错误码
- 样例报文

### 3.12 异常处理与补偿机制

至少说明：

- 幂等
- 重试
- 回查
- 补偿
- 人工介入

### 3.13 风险控制与审计要求

至少说明：

- 风控点
- 审计日志
- 操作留痕
- 权限控制

### 3.14 排期与 Git 提交节点

至少说明：

- 本系统开始日期
- 本系统阶段里程碑
- 每一批提交范围
- 每一批 Git commit 节点

## 4. 执行要求

从 `Sunday, July 19, 2026` 开始，所有系统设计都必须先对照本模板补齐后，才进入代码实现。

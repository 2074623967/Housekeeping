本文档用于在 `Monday, July 20, 2026` 这个基线时点，说明家政服务支付平台当前已经真实落地的系统、验证结果，以及下一轮继续推进的正式版改造方向。

## 4. 下一步正式版优先级

### P0：payment-core 正式版补强

优先补强：

1. 支付协议管理
2. 支付日终处理与对账前置事实收口
3. 支付任务中心与失败重试正式化
4. 真实渠道接入抽象与签名验签正式化
5. 限流、幂等、超时关闭、任务监控正式化
6. 退款中心继续补强为生产版
7. 支付核心与账户账务、清分、结算的跨系统事件边界
8. 真实退款渠道请求、退款验签、退款流水与退款差错补偿正式化

### P1：第二阶段三系统去内存态

优先顺序：

1. `accounting-system` 先落 MyBatis + MySQL
2. `clearing-system` 再落清分规则、批次、明细正式表与 Mapper
3. `settlement-system` 最后落审核、出款、回写正式表与 Mapper

### P2：跨系统联调

目标链路：

1. `payment-core` 支付成功
2. `accounting-system` 记账
3. `clearing-system` 生成清分结果
4. `settlement-system` 生成结算单
5. 出款批次发起

## 5. 当前开发策略

从这个时点开始，后续自动化推进遵循以下原则：

1. 先保证每个子系统有独立边界和第一版可运行骨架
2. 再把最关键的系统逐个从内存态升级到正式持久化实现
3. 每一轮都同时补文档、代码、测试和 Git 提交
4. 优先做最接近真实公司交付包的内容，而不是只做演示页

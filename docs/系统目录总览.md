# 系统目录总览

## 1. 文档定位

本文档用于说明 `home-service-payment-system` 仓库的目录分层规则。

## 2. 顶层目录说明

```text
home-service-payment-system
├── docs
├── systems
│   └── payment-core
├── .gitignore
└── README.md
```

说明：

- `docs`：项目级总控文档
- `systems`：按系统拆分的实现目录

## 3. 根目录 docs 的职责

根目录 `docs` 只放项目级文档，不放某一个具体系统的实现细节文档。

当前保留在根目录的文档包括：

- `整体架构与实现目标.md`
- `模块开发交付标准.md`
- `系统分期建设方案.md`
- `系统目录总览.md`

## 4. systems 目录的职责

`systems` 目录下每一个子目录代表一个独立系统。

后续建议扩展为：

```text
systems
├── payment-core
├── accounting-system
├── clearing-system
├── reconciliation-system
├── refund-center
└── deposit-system
```

## 5. 单系统目录规范

每个系统目录都应逐步统一为以下结构：

```text
system-name
├── docs
├── frontend
├── backend
└── database
```

当前 `payment-core` 已经具备：

```text
systems/payment-core
├── docs
├── frontend
└── backend
```

## 6. 当前状态

当前真正进入开发状态的系统只有一个：

- `systems/payment-core`

后面的账户账务、清结算、对账、保证金等系统，会作为并列系统逐步加入。
